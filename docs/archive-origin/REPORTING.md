# Отчеты и контроль качества трафика

## Где смотреть данные

- `clicks` — входящий трафик
- `events` — поведение и служебные события
- `leads` — заявки и статусы
- `traffic_sources_quality` — агрегированная оценка источников

## Основные показатели

- клики за день
- клики за 7 дней
- подозрительные клики
- средняя входная оценка `scoreIn`
- лиды
- качественные лиды
- плохие лиды
- продажи

## Служебная страница

- `/admin/leads`

## Отчет по источникам

- JSON: `/api/admin/reports/traffic-quality`
- таблица в `traffic_sources_quality`

## Как сравнивать кампании по сегментам

Минимальный набор сегментов для сравнения:

- `madcore_site`
- `madcore_telegram`
- `madcore_whatsapp`
- `madcore_tg_channels`

Смысл сегментов:

- `madcore_site` — основной сайтовый трафик из обычной кампании
- `madcore_telegram` — обычная кампания с акцентом на консультацию через Telegram
- `madcore_whatsapp` — обычная кампания с акцентом на консультацию через WhatsApp
- `madcore_tg_channels` — отдельная кампания с местом показа `Телеграм-каналы`

## Серверные сценарии

- `scripts/server-refresh-traffic-sources-quality.sh`
- `scripts/server-export-lead-quality-report.sh`
- `scripts/export-qualified-leads-for-yandex.sh`

## Что считать результатом рекламы

- не любой клик по кнопке
- не просто отправку формы
- а лид со статусом `qualified`
- и отдельно продажу `sale`

## Рекомендации по чтению отчета

- `усилить` — если есть качественные лиды или продажи
- `оставить` — если трафик стабилен и не деградирует
- `снизить` — если качество падает
- `отключить` — если клики идут, а лидов нет
- `проверить вручную` — если данных пока мало
