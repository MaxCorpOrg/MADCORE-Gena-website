# Безопасность проекта

## Главные принципы

- не хранить секреты в репозитории
- не публиковать служебные порты наружу
- не отправлять персональные данные в аналитику
- логировать только технически нужные данные
- использовать резервные копии и smoke-проверки

## Секреты

Разрешено хранить только:

- в `/opt/madcore-gena/.env`
- в GitHub Secrets
- в локальных вне-репозиторных файлах доступа, например `/home/max/SERVWER 2/api.txt`

Запрещено коммитить:

- `.env`
- `.env.local`
- любые другие `.env*`-файлы, кроме шаблона `.env.example`
- токены Telegram
- токены Яндекса
- пароли баз данных
- SSH-ключи
- cookies и сессионные ключи

## Защита приложения

Уже есть:

- антиспам по заявкам
- защита от дублей
- ограничение по скорости запросов
- honeypot
- разделение production и локального режима хранения

Добавлено в этом этапе:

- отдельное ограничение по скорости для `POST /api/event`
- ограничение размера тела запроса у `POST /api/event`
- закрытая служебная страница `/admin/leads` по логину и паролю
- защита `/admin/*` и `/api/admin/*` вынесена в `src/middleware.ts`
- проверка Basic Auth в `src/lib/admin-auth.ts` усилена runtime-safe декодированием, чтобы не зависеть только от `Buffer`
- `.gitignore` и `.dockerignore` усилены правилом `.env*` с явным исключением только для `.env.example`

## Что подтверждено по локальному окружению

- в рабочей директории существует локальный `.env.local`
- этот файл не отслеживается в git
- в `git ls-files` среди секретных файлов подтвержден только безопасный шаблон `.env.example`

## Защита сервера

Подтверждено в этой сессии:

- есть рабочий shell-доступ к VPS `151.247.197.153`
- для `root` дополнительно установлен вход по ключу
- production `.env` приведен к рабочему состоянию для рекламного слоя
- `docker compose ps` подтверждает healthy-состояние `app`, `nginx`, `postgres`

Целевое состояние:

- наружу открыты только `22`, `80`, `443`
- `3000`, `5432`, `matomo_db` не торчат наружу
- SSH только по ключу
- `root login` отдельно проверен и ограничен
- резервные копии выполняются регулярно
