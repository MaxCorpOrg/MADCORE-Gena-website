# Выходной фильтр качества лида

## Назначение

`LeadQualityGate` нужен для отделения:

- простого клика
- интереса к офферу
- отправленной заявки
- качественного лида
- плохого лида
- продажи

## Где живут данные

- таблица `leads`
- таблица `events`
- закрытая страница `/admin/leads`

## Какие события участвуют

- `page_view`
- `time_15_sec`
- `scroll_50`
- `scroll_90`
- `offer_view`
- `product_view`
- `telegram_click`
- `whatsapp_click`
- `call_click`
- `consultation_click`
- `form_start`
- `form_submit`
- `lead`
- `lead_notify`
- `manager_status`
- `qualified_lead`
- `bad_lead`
- `sale`

## Как считается `scoreOut`

- `+10` за `time_15_sec`
- `+10` за `scroll_50`
- `+15` за `offer_view` или `product_view`
- `+20` за `telegram_click` или `whatsapp_click`
- `+30` за `form_submit`
- `+50`, если менеджер поставил `qualified`
- `-50`, если менеджер поставил `bad`
- `100`, если есть сумма продажи

Итог ограничивается диапазоном `0–100`.

## Статусы лидов

- `new`
- `in_progress`
- `qualified`
- `bad`
- `duplicate`
- `sale`

## Служебная страница `/admin/leads`

Страница предназначена для оператора и владельца проекта.

Доступ к ней и к связанным служебным API закрыт через `src/middleware.ts`:

- защищены `/admin/*`
- защищены `/api/admin/*`
- используется Basic Auth по `ADMIN_USERNAME` и `ADMIN_PASSWORD`
- декодирование заголовка авторизации выполнено в runtime-safe виде, чтобы защита одинаково работала и в middleware-окружении, и в обычном Node.js runtime

На live уже подтверждено:

- `401` без авторизации для `/admin/leads`
- `200` с авторизацией для `/admin/leads`
- `401` без авторизации для `/api/admin/reports/traffic-quality`
- `200` с авторизацией для `/api/admin/reports/traffic-quality`

Показывает:

- клики за день
- клики за 7 дней
- подозрительные клики
- среднюю входную оценку
- лиды за день
- лиды за 7 дней
- качественные лиды
- плохие лиды
- продажи
- сводку по кампаниям и источникам

## Что делает менеджер

Менеджер может:

- поменять статус лида
- оставить комментарий
- указать сумму продажи

После сохранения система:

- обновляет запись лида
- создает `manager_status`
- при необходимости создает `qualified_lead`, `bad_lead` или `sale`
- пересчитывает `scoreOut`

## Что считать качественным лидом

Для отчетов и Яндекс.Директа качественным считается только лид со статусом:

- `qualified`
- `sale`

Простая отправка формы не считается качественным лидом.

## Как это связано с Яндекс.Метрикой

На стороне Метрики нужно разделять два уровня:

- обычные JavaScript-цели сайта:
  - `telegram_click`
  - `whatsapp_click`
  - `call_click`
  - `consultation_click`
  - `form_start`
  - `form_submit`
  - `lead`
- офлайн-конверсии после решения менеджера:
  - `qualified_lead`
  - `sale`

То есть `qualified_lead` и `sale` не считаются простыми фронтенд-событиями. Они появляются только после обработки лида и должны использоваться для рекламной оптимизации как более точный сигнал качества.
