# MADCORE Gena Yandex Metrika Setup

## Для этого проекта уже создан отдельный счетчик

Нельзя использовать счетчик исходного сайта. Для `MADCORE Gena` уже создан отдельный счетчик:

- `109282367` — `MADCORE Gena`
- текущий production-host: `madcore.site`
- старый временный домен `gena.madcore-kavkaz.ru` больше не считается рабочим доменом сайта

## Что уже заведено

- отдельный счетчик `109282367`;
- доменная привязка счетчика обновлена под `madcore.site`;
- отдельные цели:
  - `telegram_click`
  - `whatsapp_click`
  - `max_click`
  - `call_click`
  - `form_start`
  - `form_submit`
  - `lead`
- на стороне самого счетчика включены:
  - `webvisor`
  - `clickmap`
  - архивирование Вебвизора
  - отслеживание ссылок
  - точный отказ
  - `measurement_enabled`
- в интерфейсе Метрики уже сохранена первая рабочая воронка:
  - `Главная -> отправка формы`
  - шаг `1`: просмотр страницы `https://madcore.site/`
  - шаг `2`: достижение JS-цели `form_submit`
- live production реально отдает клиентский код Метрики с этим counter id;
- `METRIKA_COUNTER_ID=109282367 SITE_WWW_DOMAIN=www.madcore.site ./scripts/production-adtech-smoke.sh https://madcore.site` проходит.

## Что дополнительно было нужно для Вебвизора

Для `madcore.site` в общем ingress `nginx` добавлено условное поведение заголовка `X-Frame-Options`:

- для обычных пользователей остается `SAMEORIGIN`;
- для загрузки из доменов Яндекс.Метрики заголовок снимается.

Это нужно, чтобы Вебвизор и связанные инструменты Метрики могли открыть страницу, не ломая обычную защиту сайта от встраивания.

Текущий hero CTA больше не использует отдельное событие `consultation_click`. Вместо него работают:

- `telegram_click`
- `max_click`

Если legacy-цель `consultation_click` уже создана в самой Метрике, ее можно оставить для исторических данных, но текущий интерфейс на нее больше не опирается.

## Что записывать в `.env`

```env
NEXT_PUBLIC_YANDEX_METRIKA_COUNTER_ID=109282367
YANDEX_METRIKA_TOKEN=<токен_если_нужен_для_api>
```

## Что делать после выбора финального домена

Финальный домен уже выбран, и для текущего контура оставлен вариант без смены счетчика:

1. Счетчик `109282367` используется для production-домена `madcore.site`.
2. Старый временный домен больше не считается рабочей точкой аналитики.

## Что проверить

- счетчик подключен на текущем домене;
- в интерфейсе Метрики доступны:
  - Вебвизор
  - карта кликов
  - карта ссылок
  - посещаемость страниц
  - действия по целям;
- в разделе `Воронки` уже есть сохраненный отчет для пути `главная -> отправка формы`;
- `reachGoal` срабатывает по всем CTA и отправке формы;
- `client_id` сохраняется в tracking;
- тестовый клик через `/go` сохраняет `yclid` и UTM;
- доменная привязка счетчика не осталась только на preview-host;
- production HTML `https://madcore.site` отдает актуальный counter id `109282367`.
